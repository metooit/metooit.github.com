    Binary Search Tree

    Introduction:

    二叉搜索树上的基本操作所花费的时间与树的高度成正比。对于一个有n个结点的二叉搜索树来说， 基本操作的最坏运行时间为O(lgn)。

    1. 二叉搜索树的概念：

    • 一棵二叉搜索树是以一棵二叉树来组织的
    • 树可以用一个链表的数据结构来表示， 其中的每一个结点就是一个对象
    • 除了key 和 卫星数据 外， 每一个结点还包含了属性 left right 和 p。 如果相应的属性不存在， 那么它的值就是NIL。 根结点是树中唯一一个父指针为NIL的结点。
    • 对于任何节点x， 它的左子树中的关键字最大不能大于x.key, 右子树中的关键字最小不能小于x.key

    2. 二叉搜索树上的操作：

    2.1 如何按序打印二叉树：

    • 二叉搜索树的性质决定了我们可以通过简单的递归算法来按顺序打印出二叉树的所有关键字。这种方法称作是中序遍历（inorder tree walk） 中序遍历： 输出的子树根的关键字位于其左子树关键字和右子树关键字之间 先序遍历： 输出的根的关键字在其左右子树关键字之前 ** 后序遍历： 输出的根的关键字在其左右子树关键字之后
